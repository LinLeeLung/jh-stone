# 峻晟 ERP — 應收帳第一版規格

> 目的：將「訂單完成後如何形成可請款資料、如何依客戶規則彙整成月帳單、如何記錄收款」落成第一版可開發規格。

---

## 1. 範圍

第一版只處理以下事項：

1. 從 `salesOrders` 判定「是否可請款」。
2. 依 `customer` 的請款規則，把同客戶的可請款訂單彙整成帳單。
3. 讓會計在系統內查看帳單、調整帳單、登記收款。
4. 回寫訂單的應收摘要欄位。

第一版**不處理**：

1. 正式會計總帳分錄。
2. 與外部會計軟體的 API 串接。
3. 自動開立電子發票。
4. 複雜分期、跨月攤提、利息計算。

---

## 2. 業務規則

### 2.1 請款對象

- 請款對象一律為 `customer`。
- 業主、安裝地址、工地僅作為明細顯示，不作為應收主體。

### 2.2 請款週期規則

每個客戶需指定一種請款週期：

1. `cutoff25`
   - 預設規則。
   - 上月 26 日至本月 25 日完工的訂單，歸入下個月帳單。

2. `monthEnd`
   - 少數客戶例外規則。
   - 當月月底前完工的訂單，歸入下個月帳單。

### 2.3 可請款觸發條件

訂單可進入應收，採雙軌判定：

1. 完工型應收
   - `installedAt` 已有值。
   - 依客戶請款週期歸入對應帳期。

2. 客戶延遲型應收
   - 自 `createdAt` 起超過 3 個月仍未安裝。
   - 且未安裝責任需被標記為 `customer`。
   - 這類訂單也可形成應收。

### 2.4 金額規則

- 一張訂單通常一次請完整筆。
- 帳單內必須保留訂單細項與總和，不能只有總額。
- 是否開發票依客戶需求決定，不是固定規則。

### 2.5 收款規則

第一版支援下列收款方式：

1. 匯款 `transfer`
2. 支票 `check`
3. 切口票扣款 `cutTicketDeduction`

### 2.6 對帳規則

- 對帳主軸為：`客戶 -> 帳期內所有訂單`。
- 帳單是客戶級月結資料，不是單張訂單逐張送帳。

### 2.7 人工作業規則

- 會計保有人工調整權限。
- 允許手動移除某張訂單不併入本期帳單。
- 允許手動新增調整金額與備註。

## 3. 角色與權限

建議新增角色：`會計`

### 3.1 可見與可操作

- `admin` / `管理者`
  - 可查看、建立、調整、收款、作廢。

- `會計`
  - 可查看、建立、調整、收款、匯出。

- `員工`
  - 不可查看應收金額與收款資訊。

- `客戶`
  - 第一版不開放查看應收帳。

## 4. 資料模型

第一版採兩層模型：

1. 訂單級應收明細：`accountsReceivableItems`
2. 客戶級月帳單：`accountsReceivableBills`

### 4.1 `customers` 新增欄位

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| `billingCycleType` | enum | `cutoff25` / `monthEnd` |
| `invoicePreference` | enum | `required` / `optional` / `none` |
| `paymentTerms` | string | 例：次月請款 / 月結 30 天 |
| `paymentMethodsAllowed[]` | array | 例：`transfer`, `check`, `cutTicketDeduction` |
| `accountingNotes` | string | 會計備註 |

### 4.2 `salesOrders` 新增欄位

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| `installDelayResponsibility` | enum | `customer` / `internal` / `unknown` |
| `staleBillingEligibleAt` | timestamp | 預設為 `createdAt + 3 months` |
| `billingEligible` | bool | 是否已符合可請款條件 |
| `billingTriggerType` | enum | `installed` / `staleUninstalled` / `manual` |
| `billingEligibleReason` | string | 顯示可請款原因 |
| `receivableStatus` | enum | `none` / `pending` / `grouped` / `partial` / `paid` |
| `receivableTotal` | number | 已形成應收總額 |
| `receivedTotal` | number | 已收總額 |
| `balanceDue` | number | 未收總額 |
| `latestArBillId` | string | 最近帳單 id |
| `latestInvoiceNo` | string | 最近發票號 |

### 4.3 `accountsReceivableItems`

一筆通常對應一張訂單，是應收明細的最小單位。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| `id` | doc id | 自動產生 |
| `sourceOrderId` | string | 對應 `salesOrders.id` |
| `sourceOrderNo` | string | 對應訂單號 |
| `customerId` | string | 請款客戶 |
| `billingMonth` | string | 例：`2026-06` |
| `billingCycleType` | enum | `cutoff25` / `monthEnd` |
| `billingTriggerType` | enum | `installed` / `staleUninstalled` / `manual` |
| `billingDate` | timestamp | 實際用來歸帳期的日期 |
| `lineItems[]` | array | 從訂單金額明細快照而來 |
| `amountUntaxed` | number | 未稅金額 |
| `taxAmount` | number | 稅額 |
| `amountTotal` | number | 含稅金額 |
| `invoiceRequired` | bool | 是否需開發票 |
| `itemStatus` | enum | `pending` / `grouped` / `void` |
| `billId` | string? | 已併入哪張帳單 |
| `notes` | string | 會計備註 |
| `orderSnapshot` | object | 訂單快照 |
| `createdAt` | timestamp | 建立時間 |
| `createdByUid` | string | 建立人 |

### 4.4 `accountsReceivableBills`

一張帳單對應一個客戶在一個帳期下的請款結果。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| `id` | doc id | 自動產生 |
| `billNo` | string | 帳單號，例如 `ARB2606001` |
| `customerId` | string | 客戶 |
| `customerSnapshot` | object | 客戶名稱/統編/地址快照 |
| `billingMonth` | string | 例：`2026-06` |
| `billingPeriodStart` | date | 帳期起 |
| `billingPeriodEnd` | date | 帳期迄 |
| `cycleType` | enum | `cutoff25` / `monthEnd` |
| `itemIds[]` | array | 包含的明細 id |
| `orderIds[]` | array | 關聯訂單 id |
| `orderCount` | number | 訂單數 |
| `amountUntaxed` | number | 未稅 |
| `taxAmount` | number | 稅額 |
| `amountTotal` | number | 含稅 |
| `invoiceRequired` | bool | 本帳單是否需開票 |
| `invoiceNo` | string | 發票號 |
| `invoiceIssuedAt` | timestamp | 開票日 |
| `paymentStatus` | enum | `draft` / `issued` / `partial` / `paid` / `void` |
| `paidAmount` | number | 已收 |
| `balanceAmount` | number | 未收 |
| `payments[]` | array | 收款紀錄 |
| `adjustments[]` | array | 調整紀錄 |
| `notes` | string | 會計備註 |
| `createdAt` | timestamp | 建立時間 |
| `createdByUid` | string | 建立人 |
| `updatedAt` | timestamp | 更新時間 |
| `updatedByUid` | string | 更新人 |

### 4.5 `payments[]` 結構

```json
{
  "paidAt": "timestamp",
  "method": "transfer | check | cutTicketDeduction",
  "amount": 50000,
  "referenceNo": "可選，匯款末五碼/支票號",
  "note": "備註",
  "recordedByUid": "uid"
}
```

### 4.6 `adjustments[]` 結構

```json
{
  "type": "manualAdd | manualDeduct | writeOff | discount",
  "amount": 1000,
  "note": "調整原因",
  "at": "timestamp",
  "byUid": "uid"
}
```

## 5. 應收生成規則

### 5.1 訂單變成可請款

當符合以下任一條件時，訂單可建立 `accountsReceivableItems`：

1. `installedAt` 不為空。
2. `createdAt + 3 months <= now` 且 `installDelayResponsibility === 'customer'`。
3. 管理者或會計手動標記為可請款。

### 5.2 帳期歸屬

#### `cutoff25`

- 若完工日或應收觸發日落在 4/26 ~ 5/25，則 `billingMonth = 2026-06`。

#### `monthEnd`

- 若完工日或應收觸發日落在 5/1 ~ 5/31，則 `billingMonth = 2026-06`。

### 5.3 明細建立原則

- 一張訂單只產生一筆 `accountsReceivableItems`。
- 該筆明細保留訂單細項與金額快照。
- `chargeable=false` 的訂單不得建立應收明細。

### 5.4 帳單生成原則

- 由會計或管理者手動執行「生成本期帳單」。
- 系統按 `customerId + billingMonth + cycleType` 分組。
- 同客戶同帳期可包含多張訂單。
- 同客戶同帳期僅允許一張有效帳單；若已有非 `void` 帳單，系統不得再生成第二張。
- 若會計手動排除某筆訂單，該訂單留在 `pending`，不併入當期帳單。

## 7. 系統行為

### 7.1 自動回寫訂單摘要

當帳單或收款變更時，自動回寫來源訂單：

- `receivableTotal = sum(相關帳單 amountTotal, 排除 void)`
- `receivedTotal = sum(相關帳單 paidAmount, 排除 void)`
- `balanceDue = receivableTotal - receivedTotal`
- `receivableStatus` 依上述結果推導

### 7.2 開票欄位

- 帳單可記錄 `invoiceNo`、`invoiceIssuedAt`。
- 第一版不自動產生發票，只提供人工回填。

### 7.3 收款邏輯

- 允許一次收清。
- 允許部分收款。
- 允許多次收款。
- 允許切口票扣款被記為正式收款方式。

## 9. Cloud Functions / Service 規劃

第一版建議新增：

1. `computeBillingEligibility`
   - 重新計算訂單是否符合可請款條件。

2. `generateReceivableItems`
   - 由符合條件的訂單建立應收明細。

3. `generateReceivableBills`
   - 依帳期把明細彙整成客戶帳單。

4. `recordReceivablePayment`
   - 對帳單新增收款紀錄並回寫摘要。

5. `syncOrderReceivableSummary`
   - 帳單變動時回寫 `salesOrders` 應收摘要。

## 10. 第一版開發順序

### P1 資料層

1. `customers` 追加會計欄位。
2. `salesOrders` 追加應收摘要欄位。
3. 建立 `accountsReceivableItems` / `accountsReceivableBills`。

### P2 流程層

1. 計算可請款條件。
2. 建立應收明細。
3. 依客戶與帳期生成帳單。

### P3 UI

1. 訂單詳情應收區塊。
2. 應收明細列表。
3. 月帳單列表與詳情。

### P4 收款

1. 收款登錄。
2. 部分收款。
3. 自動回寫摘要。

## 11. 仍待確認事項

1. 「切口票扣款」是否需要另記關聯單號或來源單號。
2. 帳單號 `billNo` 編碼規則是否採年月流水。
3. 同一客戶同帳期已確定僅保留一張有效帳單；若日後要支援多張帳單，需另行設計分帳單鍵值。
4. 若訂單金額在應收明細建立後又變更，是否允許同步更新，或需先作廢舊明細再重建。
5. 發票需求是每張帳單單一值，還是帳單內可混合「部分訂單開票、部分不開票」。
