# 薪資計算說明

> 適用系統：JH-Stone 人資管理系統  
> 計算引擎：Firebase Cloud Function `calculatePayroll`

---

## 一、加班費

### 資料來源

從 Firestore `overtimeRequests` collection 查詢該員工**當月**且**已雙層主管簽核**（`status: "approved2"`）的加班申請，每筆紀錄包含 `date`、`hours`。

### 前提條件

- 員工必須有 Firebase 帳號（`uid`）
- 加班申請需完成兩層審核

### 計算方式（勞基法第24條）

時薪基準（月薪制）：

$$
\text{時薪} = \frac{\text{月薪}}{240} \quad (30\text{ 天} \times 8\text{ 小時})
$$

每筆加班申請個別計算：

| 加班時數    | 費率       |
| ----------- | ---------- |
| 前 2 小時   | 時薪 × 4/3 |
| 第 3 小時起 | 時薪 × 5/3 |

$$
\text{加班費} = \begin{cases} h \times \text{時薪} \times \dfrac{4}{3} & h \leq 2 \\[6pt] 2 \times \text{時薪} \times \dfrac{4}{3} + (h-2) \times \text{時薪} \times \dfrac{5}{3} & h > 2 \end{cases}
$$

### 完整流程

員工 App 提交加班申請 → 主管兩次審核通過 → 點「計算薪資」→ Cloud Function 自動加總計入

## 二、請假扣薪

### 資料來源

從 Firestore `leaveRequests` collection 查詢該員工**當月**且**已雙層主管簽核**（`status: "approved2"`）的請假申請，每筆紀錄包含 `type`（假別）、`unit`（天/小時）、`days` 或 `hours`。

### 扣薪規則

| 假別         | 扣薪比例          |
| ------------ | ----------------- |
| 年假（特休） | **不扣薪**        |
| 病假         | **半薪**（× 0.5） |
| 事假、其他   | **全扣**（× 1.0） |

單位換算：天 → × 8 小時

$$
\text{扣薪} = \text{時薪} \times \text{請假時數} \times \text{比例}
$$

### 遲到 / 早退扣薪（另計）

從打卡紀錄（`attendance`）比對上下班時間，超過系統設定的寬限分鐘數後按分鐘扣薪。規則可於「系統設定 → 差勤規則」調整。

## 三、新進員工未滿月計薪

### 適用條件

- 薪資類型為**月薪制**
- 員工到職日（`startDate`）在當月內

> 時薪制、日薪制本來就按實際工時計算，不受此規則影響。

### 計算方式

$$
\text{當月底薪} = \text{月薪} \times \frac{\text{到職日至月底天數}}{\text{當月總天數}}
$$

**範例**：5 月 15 日到職，月薪 40,000 元

$$
40{,}000 \times \frac{17}{31} \approx 21{,}935 \text{ 元}
$$

### 備註

- 到職月按比例計算，**次月起恢復全額**
- 薪資明細同時儲存：
  - `baseSalary`：本月實領底薪（按比例後）
  - `baseSalaryFull`：員工原始月薪

## 五、實領薪資公式

$$
\text{實領} = \text{底薪} + \text{獎金} + \text{加班費} + \text{伙食費} - \text{請假扣薪} - \text{遲到早退扣薪} - \text{曠職扣薪} - \text{勞保費} - \text{健保費} - \text{眷屬健保費} - \text{便當費} - \text{房租} - \text{水費} - \text{電費} - \text{體檢費} - \text{服務費} - \text{借款本金} - \text{借款利息}
$$

最小值為 **0**（不會出現負數）。

## 六、曠職扣薪

### 定義

曠職 = 工作日**無出勤打卡**且**無核准請假**。

### 自動偵測條件

Cloud Function 計薪時，逐一檢查當月每一天，同時符合以下**全部條件**才計入曠職：

| 條件 | 說明 |
|------|------|
| ① 非未來日期 | 計算當天之後的日期不納入 |
| ② 非到職前 | 員工到職日（`startDate`）之前不計 |
| ③ 非週末 | 週六、日自動跳過（補班日除外，見第八節） |
| ④ 非國定假日 | 需在「系統設定 → 國定假日」中設定 |
| ⑤ 無出勤打卡 | `attendance` 集合中找不到 `punchIn` 記錄 |
| ⑥ 無核准請假 | `leaveRequests` 中無 `approved2` 的核准假單涵蓋該日 |

### 扣薪計算

$$
\text{日薪} = \text{時薪} \times 8 = \frac{\text{月薪}}{240} \times 8
$$

$$
\text{曠職扣薪} = \text{日薪} \times \text{曠職天數}
$$

### 儲存欄位

| 欄位 | 說明 |
|------|------|
| `absentDays` | 曠職天數 |
| `absentDeduction` | 扣薪金額（元） |
| `absentDetail` | 各曠職日期陣列（`YYYY-MM-DD`） |

## 七、兩次發薪（5 日 / 10 日）

系統將每月薪資拆為兩筆：

### 5 日實發（`firstPayment`）

以**投保薪資**為底，加計**申報加班費**，扣除所有固定費用及曠職：

$$
\text{5日實發} = \max\!\left(0,\; \text{投保薪資} + \text{申報加班費} - \text{勞保} - \text{健保} - \text{眷屬健保} - \text{便當費} - \text{住宿費} - \text{水費} - \text{電費} - \text{體檢費} - \text{服務費} - \text{借款本金} - \text{借款利息} - \text{曠職扣薪}\right)
$$

### 10 日補發（`secondPayment`）

$$
\text{10日補發} = \text{實領} - \text{5日實發}
$$

> 若底薪高於投保薪資、或當月加班費較多，差額於 10 日補齊。  
> 若 5 日已多發（如曠職未及時計入），10 日補發會為負數（即補扣）。

## 八、國定假日與補班日

兩者均在「**系統設定**」中維護，支援手動新增及一鍵自動載入（資料來源：[ruyut/TaiwanCalendar](https://github.com/ruyut/TaiwanCalendar)）。

### 國定假日（`publicHolidays`）

- 格式：`{ date: "YYYY-MM-DD", name: "假日名稱" }`
- 效果：計薪時該日**不計曠職**

### 補班日（`makeupWorkdays`）

- 格式：`{ date: "YYYY-MM-DD", name: "補班" }`  
- 效果：即使是週六/日，計薪時仍視為**工作日**；無打卡且無請假 → 計曠職

### 邏輯優先順序

```
該日 → 是補班日？
  ├─ 是 → 當作工作日 → 繼續偵測
  └─ 否 → 是週末？ → 跳過

繼續偵測 →
  是國定假日？ → 跳過
  有打卡？ → 跳過
  有核准請假？ → 跳過
  → 計為曠職
```
